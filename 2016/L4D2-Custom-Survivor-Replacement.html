开始准备

好的，让我们从最基础的开始。首先你需要安装 Steam 和 Left 4 Dead 2. 额外的我们还需要安装 Left 4 Dead 2 Authoring Tools 和 Source SDK. 你点击Seam图标，选择 库，然后从这 库 的菜单中点击 工具（如果你默认没有显示它们的话这会显示所有可用的工具）。

现在点击你所要的工具（Left 4 Dead 2 Authoring Tools 和 Source SDK）然后选择安装游戏。接着在弹窗里确认安装。在你做这些的时候我准备去玩几局昆特牌:) 因为昆特牌实在是太xx爽了。你们弄好了我就会回来。

OK，节目继续，基础的准备都准备好了，让我们将每个程序运行一次让它们初始化。如果你没有在安装时创建快捷方式那你能在Steam的库里运行它们。我们先运行 Authoring Tools… 如果没有意外的话它启动后基础的界面应该是酱紫的:

注意你现在也将 Face Poster 和 Model Viewer 安装上了。那些现在对你都不重要。
现在让我们关闭这个程序然后启动 Source SDK. 第一次启动你会看见一个”copying files please wait…”的窗口，耐心等待它结束。当它运行完毕后会打开一个浏览窗口还有一些我压根不关心的公告，直接右上角点X让它滚蛋…现在的界面和 Authoring Tools 非常的相似:

请注意 Source SDK 里的 Model Viewer 和 Face Poster 与 Authoring Tools 里的 不完全相同。这个东西你暂时还用不着关心。现在只需要记住就好了。 OK, 现在我们所需要的东西都安装运行的不错了，那我们继续。

模型处理

OK，如果我们想对模型进行修改我们需要能够打开这些模组和各种各种样的模组文件格式。还需要将它们导出和导入我们的游戏。那我们就先把埃利斯从求生之路2里提取出来开始。

首先我们需要 GCFScape 这个程序。
这里是下载链接:http://nemesis.thewavelength.net/files/files/gcfscape183.exe

这是个可执行文件，但我保证它不是病毒。如果你不相信你可以自己百度这个链接来满足你的妄想症(原文为If you don’t trust me you can go Google the bloody link yourself Captain Paranoia.)。运行安装程序并让它自己选择默认安装选项(这样是为了让我们的安装的环境都一样，别到时候哭着来说自己和我的不一样做不了，因为你非要买萌地把它安装在 “你个坑货” 文件夹里)。(原文为:so don’t come crying to me when something doesn’t work because you wanted to be cute and install it in the “FudgePuddles” folder on your computer.)

一旦它安装好了你就在 开始菜单->所有程序->Nem`s Tools->GCFScape 运行它。打开后大概就像这样:

OK, 然后让我们把那个乡巴佬(redneck 乡下人 这里指埃利斯)抓出来。点击 文件(File)->打开(Open) 或者点击左上角的文件夹图标。我们需要打开保存求生之路2里人物模型VPK文件的文件夹(你可以把VPK想象成一个ZIP压缩文件)。

这些文件的路径就像这样:
C:\Program Files\Steam\steamapps\common\left4dead2\left4dead2\pak01_dir.vpk

注:每个人的路径可能不一样，你需要到安装求生之路2时的路径里寻找

找到的文件如图所示:

选择这个文件(pak01_dir.vpk)然后点击打开(Open)，你能得到这东西:

这些就是这个VPK文件包含的所有东西了。你能在里面找到所有的地图模型、纹理脚本和别的好东西。我们将在这里把 埃利斯提取出来。这个文件相当重要，对以后你自己做VPK插件展示了正确的文件结构，返回去再看一次这个文件结构吧。

我们继续…在左边的窗口里，点击 models 旁边的加号( + )，往下滑再点击 survivors 旁边的加号。在右边的窗口里你能看见许多文件。如果你在右边窗口里寻找你能看见一些叫做”survivor_machanic”的文件，那个就是我们亲爱的埃利斯。接着我们酱紫做：

  1. 让我们在右边的窗口里将这些文件标出来(你能得到五个这样的文件)。
  2. 右键选中它们然后选择提取(Extract)，它会询问你解压到哪。让我们养成一个好习惯把它们都放在一个合适的位置…那点击桌面。
  3. 新建文件夹，把它命名为”survivor_ellis”然后标记它。
  4. 再次新建文件夹，命名为”models”。
  5. 再次新建文件夹，命名为”survivors”
  6. 选中这个文件夹点击OK。

我们刚才创建了一个和VPK文件相似结构的文件夹，因为在VPK文件里这些文件都是在 models\survivors\ 这个文件夹里。有感觉了么？我实在是词穷不知道怎么解释这个了，直接看提取后的路径图吧:

现在，庆祝一下吧，让我们把埃利斯的表面材质纹理文件提取出来(let’s grab Ellis’s main texture files. )。

  • 返回GCFScape程序，这次我们点击materials旁边的加号。
  • 然后点击models旁边的加号。
  • 再点击survivors旁边的加号。
  • 在右边的窗口你能看见一些文件夹，注意器中一个叫做 mechanic, 右键它选择提取。
  • 这次还是会询问你提取到哪个地方，同样的我们照着VPK文件的结构放置它。
  • 于是我们将它保存在survivor_ellis\materials\models\survivors(将mechanic文件夹放在survivors文件夹里，这样mechanic文件夹里所有的文件都在里面)。
  • 这样我们就得到了我们所需要埃利斯所有的模型和纹理文件。
  • 关闭 GCFScape程序。

反编译

现在我们已经有了模型文件，我们将把它们转换成能进行修改的文件格式。这就意味着我们需要反编译它们。具体说是需要反编译 survivor_mechanic.mdl 文件，它包含了我们所需要的实际网格数据。

为了反编译这些文件我们需要别的软件(啊啊啊啊啊啊啊!)，这次我们需要的程序叫做 MDLDecompiler.
你能在这里下载这个版本：Hooch`s MDL Decompiler[www.hl2.boo.pl](链接失效)

下载它到你的桌面然后解压文件(RAR文件和ZIP文件差不多，如果没有的话WinRAR足够胜任)。
解压后你应该得到一个可执行文件，一个readme和另一个RAR文件。
新的RAR压缩文件里有基本的安装说明，现在我们就将跟着它走。

复制 MDLDecompiler Fixed.exe 到下面的文件夹：
C:\Program Files\steam\steamapps\Common\sourcesdk\bin\ep1\bin

注：这个目录指的是你安装Source SDK工具时选择的文件夹。

  • 现在右键这个exe文件选择 发送->桌面(创建快捷方式) 这样我们想运行它时就不用每次都刨这么多的文件夹了。
  • 然后双击这个快捷方式(或者这个exe文件如果你还没有关闭窗口的话)，NDKDecompiler就启动了。

注意有个那有一个”修正版(fixd)”。理论上说应该修成后的版本运行会很好而且更简单，但是时代在变迁啊。这个时启动画面：

在它显示选择模型文件(Choose Model File)的右边点击那三个点，你就能浏览埃利斯的MDL文件，这个文件位于下面的这个路径里：
survivor_ellis\models\survivors\mechanic_ellis.mdl

  • 在选中mdl文件后点击OK，然后点击选择输出文件夹(Choose Output Directory)右边的三个点。
  • 你可以提取到任何地方，但是为了让事物井然有序，我们会将它反编译到survivor_ellis文件夹里。
  • 那在survivor_ellis文件夹双击models，再双击survivors，然后新建一个文件夹命名为”decompile”选中它。
  • 点击选择文件夹它应当返回程序主界面，确保你没有选中左边的任何一个选项然后点击提取(Extract)按钮。

这时我的程序崩溃了，我还对此没有办法，这是个非常普遍的问题。我把它卸载这个教程里是因为我认为它是一个很普遍的问题，值得一提。

从这里下载另一个版本的 mdldecompiler ：
MDLDecompiler Version 0.4.1[www.chaosincarnate.net]

我看到了一个0.5的版本，但它好似还是崩溃掉了。就用还不错的0.4.1这个老版本吧。把它复制到与之前那个版本程序一样的地方，然后再在你的桌面上新建快捷方式。运行它，再次点击选择文件可能你会得到这样一个”欢迎”界面：

  • 点击 OK 程序会自动退出。
  • 双击打开它又会出现。
  • 在左下角取消选中”Use Steam File Access”再点击选择MDL文件那三个点。
  • (这次它应该正常运行，你能像之前那样浏览survivor_mechanic.mdl文件)
  • 点击选择输出文件夹再选择之前那个decompile文件夹。
  • 现在点击提取(Extract)然后祈祷吧…然而无法加载模型（Unable to load model).

但是别担心！因为我也遇到了(╯‵□′)╯︵┻━┻。这也是一个很普遍的问题。我们还需要手动做一些别的事，那个”修正版”会自动做这些事。那就开始吧…
手动做的话我们需要一个 Notepad++ ，它能节省你大量的时间免得你被这些事逼得出心脏病，而且它比那个古老的记事本好用多了。

你能在这里下载：
Notpad ++[download.tuxfamily.org]

下载后用它默认的选项安装就好，当你点击完成按钮后它会自己启动，从我们的survivor_ellis文件夹里打开你的 survivor_mechanic.mdl文件(它在models\survivors里)。稍等一下Notepad++会将它打开，你能看到和这个差不多的东西：

需要我们关注的地方只有一小丢丢，在上面写着 IDST1 的地方。

我们将用 , 代替那个 1 。
将 1 选中然后按下键盘上的 , 键。瞧！点击保存按钮或者点 文件(File)->保存(Save) ，等待它保存后就可以退出程序了。

返回MDLDecompiler程序，它应该还是运行的，所有的东西都弄好了，再次点击提取按钮…它开始工作然后又抛出另一个文件丢失的错误。

点击OK然后这个程序貌似很可能崩溃掉，再次打开并确保你的提取目录和输出文件夹都是和以前选择的一样，我知道这样挑♂逗♂你蛮让人泄气的，但相信我现在让你经历这些问题肯定是好的，让这么多琐事扰头，以后再发生类似的事，你可能会大叫：“哦♂耶♂…这些东西在教学里都有类似的，我还解决过…我这次必定能弄好的。”那时你就明白为何我要这么做了:)
无论如何，继续吧…

现在我们需要去修复我们的mdl文件。

  • 保持MDLDecompiler程序运行，我们打开survivor_ellis文件夹，进入 models\survivors\
    文件夹找到我们的模型文件。
  • 你能看见一个叫survivor_mechanic.dx90.vtx的文件。
  • 选中它，右键选择复制，然后在当前文件夹再次右键选择粘贴。
  • 这会出现一个 survivor_mechanic.dx90 – Copy.vtx 的新文件。
  • 选中它右键，选择重命名。
  • 重命名为：survivor_mechanic.dx80.vtx ，然后回车确认这次修改。
  • 如果它询问你确认的话点击YES(蠢萌的windows)。

OK，现在让我们回到 MDLDecompiler，然后再点击那个该死的提取(Extract)按钮…它报告了一些和加载模型的东西…点击OK…如果一切安好，它会报告完成反译模型完成。当这个是听天由命了…它有时候还!会!崩溃但有时又不会。我有两台电脑，其中一台运行它没有任何问题，但另一台间歇性崩溃掉。

那台运行这个程序间歇性崩溃掉的电脑在用了一个MDLDecompiler更老的版本后能正常使用了，你能在这里下载到：http://www.mrlanky.com/mdldecompiler4.exe
